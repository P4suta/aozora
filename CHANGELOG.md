# Changelog

All notable changes to aozora are recorded in this file. The format
follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/);
the project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

This file is updated by `git-cliff` (`just changelog`) at every
release. Day-to-day commit narrative lives in `git log`; this file
records only what is observable from the outside (added features,
breaking changes, bug fixes that affect users).

## [0.2.4] - 2026-04-29

### Build
- Release matrix slimmed to three platforms
  (`x86_64-unknown-linux-gnu`, `aarch64-apple-darwin`,
  `x86_64-pc-windows-msvc`); `x86_64-apple-darwin` Intel and
  `x86_64-unknown-linux-musl` dropped.

## [0.2.3] - 2026-04-29

### Build
- Release matrix preliminary slim (superseded by v0.2.4).

## [0.2.2] - 2026-04-29

### Fixed
- **release**: explicit `rustup target add` in `release.yml` for
  the musl build target.

## [0.2.1] - 2026-04-29

### Fixed
- **release**: corrected published crate name in `release.yml`
  (`aozora-cli`, not `afm-cli`).

## [0.2.0] - 2026-04-29

### Added
- Top-level `aozora` facade crate exposing `Document` /
  `AozoraTree` / `Diagnostic`.
- WASM driver (`aozora-wasm`) with `wasm-bindgen` exports.
- C ABI driver (`aozora-ffi`) with opaque-handle + JSON diagnostics.
- Python binding (`aozora-py`) via PyO3 / maturin.
- Borrowed-arena AST (`bumpalo`) — `O(n)` walks, single-free drop.
- SIMD multi-pattern scanner (`aozora-scan`) — Teddy / Hoehrmann DFA
  / memchr backends.
- Eytzinger sorted-set lookup (`aozora-veb`).

### Changed
- Workspace split into 18 focused crates from the original
  monolithic layout.

## [0.1.0] - 2026-04-15

### Added
- Initial public preview. CLI (`check` / `fmt` / `render`),
  seven-phase lexer, HTML and canonical serialiser, structured
  diagnostics, Shift_JIS + 外字 resolver, Aozora Bunko corpus
  sweep.
